\chapter{}
\section{Introducción}

Este informe documenta el despliegue de una aplicación web dinámica basada en PHP y MariaDB, ejecutada mediante contenedores Docker y orquestada con Docker Compose. El objetivo del laboratorio fue implementar un stack web reproducible, capaz de separar la lógica de aplicación, la base de datos, la persistencia de información y la comunicación entre servicios.

La solución desarrollada cumple con las operaciones CRUD solicitadas: crear, leer, actualizar y eliminar registros desde una interfaz web. Para ello, se utilizó un contenedor de aplicación construido desde un \texttt{Dockerfile}, un contenedor de base de datos basado en una imagen oficial de MariaDB, un volumen persistente y una red virtual administrada por Docker Compose.

\newpage
\section{Desarrollo}

La arquitectura se compone de dos servicios definidos en el archivo \texttt{docker-compose.yml}. El servicio \texttt{app} corresponde a la aplicación web PHP con Apache, mientras que el servicio \texttt{db} ejecuta MariaDB. Ambos servicios se comunican a través de la red \texttt{laboratorio\_net}, y la base de datos conserva su información mediante el volumen \texttt{db\_data}.

\begin{table}[H]
\centering
\begin{tabular}{|p{3.4cm}|p{10.5cm}|}
\hline
\textbf{Componente} & \textbf{Descripción} \\
\hline
\texttt{app} & Contenedor PHP 8.3 con Apache. Incluye el código de la aplicación y la extensión \texttt{pdo\_mysql} para conectarse a la base de datos. \\
\hline
\texttt{db} & Contenedor MariaDB 11.4 basado en imagen oficial. Inicializa la tabla de contactos mediante \texttt{db/init.sql}. \\
\hline
\texttt{db\_data} & Volumen persistente que conserva la información aunque los contenedores sean detenidos o recreados. \\
\hline
\texttt{laboratorio\_net} & Red virtual tipo \textit{bridge} que permite aislar la comunicación entre la aplicación y la base de datos. \\
\hline
\end{tabular}
\caption{Componentes principales del stack Docker implementado.}
\end{table}

\section{Construcción}

La construcción de la imagen de la aplicación se realizó con el comando:

\begin{lstlisting}[caption={Construcción de la imagen de aplicación}]
docker compose build
\end{lstlisting}

Este comando lee el \texttt{Dockerfile} ubicado en la carpeta \texttt{app/}. Durante el proceso se utiliza la imagen base \texttt{php:8.3-apache}, se instala la extensión \texttt{pdo\_mysql} y se copia el archivo \texttt{index.php} al directorio de publicación de Apache.

\section{Ejecución}

Una vez construida la imagen, los contenedores se levantaron en segundo plano mediante:

\begin{lstlisting}[caption={Ejecución de los servicios}]
docker compose up -d
docker ps
\end{lstlisting}

El parámetro \texttt{-d} permite ejecutar los servicios en modo desacoplado. Posteriormente, el comando \texttt{docker ps} verifica que los contenedores \texttt{laboratorio\_app} y \texttt{laboratorio\_db} estén activos. En esta etapa también se confirma la publicación del puerto \texttt{8080}, que redirige hacia el puerto \texttt{80} del contenedor web.

\section{Configuración Docker}

\subsection{Dockerfile}

El archivo \texttt{app/Dockerfile} define la imagen de la aplicación. Se utilizó una imagen oficial de PHP con Apache, a la cual se le agregó la extensión \texttt{pdo\_mysql}. Esta extensión permite que la aplicación PHP se conecte a MariaDB usando PDO.

\begin{lstlisting}[caption={Dockerfile de la aplicación}]
FROM php:8.3-apache

RUN docker-php-ext-install pdo pdo_mysql

WORKDIR /var/www/html
COPY index.php /var/www/html/index.php

EXPOSE 80
\end{lstlisting}

\subsection{Docker Compose}

El archivo \texttt{docker-compose.yml} vincula los servicios \texttt{app} y \texttt{db}. La aplicación expone el puerto \texttt{8080} hacia el navegador, mientras que la base de datos se mantiene dentro de la red interna y utiliza variables de entorno para definir usuario, contraseña y nombre de la base de datos.

\begin{lstlisting}[caption={Comandos principales de despliegue}]
cp .env.example .env
docker compose build
docker compose up -d
docker ps
docker logs laboratorio_db
\end{lstlisting}

\subsection{Persistencia y aislamiento}

La persistencia se implementó mediante el volumen \texttt{db\_data}, montado sobre \texttt{/var/lib/mysql}. Gracias a esto, los datos creados desde la aplicación se conservan incluso si los contenedores son detenidos o recreados. El aislamiento se logró mediante la red virtual \texttt{laboratorio\_net}, evitando exponer innecesariamente el servicio de base de datos fuera del entorno Docker.

\section{Conclusión}

La implementación del laboratorio permitió integrar una aplicación web dinámica en PHP con una base de datos MariaDB desplegada mediante Docker, cumpliendo con los requisitos de CRUD, persistencia y aislamiento de servicios. El uso de \texttt{docker compose} simplificó la orquestación del entorno y facilitó reproducir el sistema de manera consistente en distintos equipos.

Durante el desarrollo se comprobó que la separación entre el contenedor de aplicación y el contenedor de base de datos mejora el mantenimiento de la solución, porque cada servicio cumple una responsabilidad específica y se comunica a través de una red virtual controlada. Además, el uso de volúmenes permitió conservar la información aun cuando los contenedores fueron reiniciados o recreados.

En términos funcionales, la interfaz web entrega un flujo claro para crear, leer, actualizar y eliminar registros, lo que valida el objetivo principal del laboratorio.
